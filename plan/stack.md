# Stack Tecnológico

## Frontend
- **Framework**: Flutter 3.x
- **Lenguaje**: Dart
- **UI**: Material Design 3
- **State Management**: Riverpod
- **Routing**: go_router
- **Plataformas**: Web + iOS + Android

## Backend & Database
- **Backend as a Service**: Supabase
- **Database**: PostgreSQL
- **Authentication**: Supabase Auth (OAuth: Google, GitHub)
- **Real-time**: Supabase Realtime
- **Edge Functions**: Deno/TypeScript
- **MCP Integration**: Supabase MCP Server

## APIs Externas
- **AI Service**: OpenAI API (GPT-4)
- **Streaming**: Server-Sent Events o streaming HTTP para respuestas en tiempo real

## Paquetes Flutter Clave
- **Supabase**: supabase_flutter
- **Riverpod**: flutter_riverpod, riverpod_annotation
- **Routing**: go_router
- **HTTP**: dio (para llamadas a Edge Functions)
- **Markdown**: flutter_markdown
- **Icons**: Material Icons (incluido en Flutter)

## DevOps & Tools
- **Package Manager**: pub
- **Version Control**: Git
- **Deployment Web**: Firebase Hosting / Vercel / Netlify
- **Deployment Mobile**: App Store / Google Play
- **Linting**: flutter analyze, dart analyze
- **Formatting**: dart format

## Arquitectura de Desarrollo
- **Multiplataforma**: Un solo código para web y mobile
- **Responsive**: MediaQuery y LayoutBuilder para adaptación
- **Type Safety**: Dart con null safety
- **Code Generation**: Para Riverpod providers (opcional)
